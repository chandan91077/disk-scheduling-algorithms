About Disk Scheduling

What is Disk Scheduling?

Disk scheduling is done by operating systems to schedule I/O requests arriving for the disk. The goal of disk scheduling algorithms is to minimize the seek time and maximize the disk bandwidth.

Why is it Important?

Disk scheduling is crucial because the speed of the disk system is much slower compared to the main memory and CPU. Efficient scheduling can significantly improve overall system performance.

Disk Scheduling Algorithms

FCFS

First Come First Serve - Requests are served in the order they arrive.

Advantages: Simple, fair
Disadvantages: Poor performance

SSTF

Shortest Seek Time First - The closest request is served next.

Advantages: Better performance than FCFS
Disadvantages: Starvation possible

SCAN

Elevator Algorithm - The disk arm moves in one direction, serving requests along the way.

Advantages: No starvation, good performance
Disadvantages: Long wait for some requests

C-SCAN

Circular SCAN - The arm moves in one direction, then jumps back to the start.

Advantages: More uniform wait time
Disadvantages: More movement than SCAN

LOOK

Similar to SCAN but only goes as far as the last request in each direction.

Advantages: Less movement than SCAN
Disadvantages: Still has some inefficiency

C-LOOK

Similar to C-SCAN but only goes as far as the last request before reversing.

Advantages: Most efficient of all
Disadvantages: More complex to implement
